# GKCII 环境配置手册

## 项目概述

本项目为基于 **ROS (Robot Operating System)** 的多机器人仿真系统，包含两个 catkin 工作空间：

- **catkin_ws** — 主仿真环境，包含 Gazebo 场景、无人机(UAV)与无人车(UGV)仿真、Velodyne 激光雷达、雷达插件、点云处理、YOLO 目标检测、AprilTag 识别等
- **damad_ws** — 多车多无人机协同控制，包含摄像头、键盘/G29 方向盘控制、二维码跟随、巡线等

## 2. 安装 ROS Noetic

```bash
# 添加 ROS 源
sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" > /etc/apt/sources.list.d/ros-latest.list'
sudo apt-key adv --keyserver 'hkp://keyserver.ubuntu.com:80' --recv-key C1CF6E31E6BADE8868B172B4F42ED6FBAB17C654

# 更新并安装
sudo apt update
sudo apt install ros-noetic-desktop-full

# 初始化 rosdep
sudo rosdep init
rosdep update

# 环境变量
echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
source ~/.bashrc

# 安装 catkin 构建工具
sudo apt install python3-catkin-tools python3-catkin-pkg
```

---

## 3. 安装 PX4 Autopilot (无人机仿真)

项目中的 `uav_cruise.sh`、`2cars.sh` 等脚本调用了 PX4 启动文件，需要安装 PX4 SITL 仿真环境。

```bash
# 安装依赖
sudo apt install -y git wget gnupg2 lsb-release software-properties-common

# 克隆 PX4 固件
cd ~
git clone https://github.com/PX4/PX4-Autopilot.git --recursive
cd PX4-Autopilot

# 安装 PX4 依赖 (Gazebo Classic)
bash Tools/setup/ubuntu.sh --no-nuttx --no-sim-tools

# 编译 Gazebo Classic 仿真
make px4_sitl_default gazebo-classic

# 设置环境变量 (添加到 ~/.bashrc)
echo "source ~/PX4-Autopilot/Tools/setup_gazebo.bash ~/PX4-Autopilot ~/PX4-Autopilot/build/px4_sitl_default" >> ~/.bashrc
echo "export ROS_PACKAGE_PATH=\$ROS_PACKAGE_PATH:~/PX4-Autopilot" >> ~/.bashrc
echo "export ROS_PACKAGE_PATH=\$ROS_PACKAGE_PATH:~/PX4-Autopilot/Tools/simulation/gazebo-classic/sitl_gazebo-classic" >> ~/.bashrc
source ~/.bashrc
```

## 13. 常见问题

### Q1: catkin_make 报错找不到某个 ROS 包
运行 `rosdep install --from-paths src --ignore-src -r -y` 自动安装缺失依赖。

### Q2: PX4 仿真启动失败
确保已正确 source PX4 环境：
```bash
source ~/PX4-Autopilot/Tools/setup_gazebo.bash ~/PX4-Autopilot ~/PX4-Autopilot/build/px4_sitl_default
```

### Q3: Gazebo 模型缺失
将项目中的自定义模型复制到 Gazebo 模型目录：
```bash
cp -r ~/GKCII/catkin_ws/models/* ~/.gazebo/models/
```

### Q4: pyapriltags 安装失败
```bash
sudo apt install -y cmake libopencv-dev
pip3 install pyapriltags --no-cache-dir
```

### Q5: yolo_detect.py 报错找不到权重文件
确认权重文件路径，修改 `yolo_detect.py` 中的 `weight_path` 变量指向正确的 `.pt` 文件位置。
